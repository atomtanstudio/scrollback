I've been benchmarking different vector databases for production RAG pipelines and the results surprised me. Here's what I found after testing 6 different solutions over 3 months with real production traffic...
SL
Sarah Lin @sarahlin_dev 2:14 PM
First up: pgvector. If you're already running Postgres, this is the easiest option. Performance was solid up to ~1M vectors with HNSW indexing. The killer feature is being able to join vector search results with your relational data in a single query. No separate service to manage.

Sarah Lin @sarahlin_dev 2:16 PM
The real surprise was Qdrant. At 5M+ vectors it was 3-4x faster than pgvector on p99 latency, and the filtering capabilities are much more powerful. You can do complex boolean filters on payload fields without sacrificing search speed.

The hybrid search mode (sparse + dense vectors) gave us a 23% improvement in retrieval quality for technical documentation queries. Worth the operational complexity if you're at scale.

TL;DR: Use pgvector if you have <1M vectors and want simplicity. Use Qdrant if you need scale, complex filtering, or hybrid search. Pinecone is fine but overpriced for what you get. Weaviate has the best DX but worst p99 latency. Chroma is only good for prototyping.
